# Antigravity Repository Instructions - EqManager

These instructions are mandatory for any Antigravity session in this repository.

## 1. Auto-Deployment Protocol
At the natural end of any implementation session (i.e., whenever code was modified and the user is being asked to test), you MUST:
1. Locate and run `deploy.ps1`.
2. Use the command: `powershell.exe -ExecutionPolicy Bypass -File .\deploy.ps1`.
3. Follow the reporting protocol in [.antigravity/DeploySkill.md](file:///i:/repos/EqManager/.antigravity/DeploySkill.md):
   - Start your confirmation with: ✅ **Deployed successfully — you can now reload your UI to test.**
   - Remind the user to reload their UI (e.g., `/reload`).
   - Summarize the specific changes made.

## 2. Commit Generation
When the user asks to "prepare a commit" or uses `/commit`, use the [CommitGeneratorSkill.md](file:///i:/repos/EqManager/.antigravity/CommitGeneratorSkill.md) in the `.antigravity` directory.
